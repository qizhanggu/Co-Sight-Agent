结果提交示例是用户需要上传的压缩包参考形式，建议至少包含：
1. `result.jsonl`
2. 基于 Co-Sight 二次开发的完整源码压缩包

`result.jsonl` 文件说明：
- 文件为 JSON Lines 格式，每行是一个独立 JSON 对象，对应一个题目的作答结果。
- 参考行示例：
{"task_id":"c61d22de-5f6c-4958-a7f6-5e9707bd3466","Question":"...","model_answer":"egalitarian","reasoning_trace":"1. ..."}

字段含义与必选性：
- `task_id`（必填）：题目唯一标识，用于和赛题进行一一对应。
- `model_answer`（必填）：最终答案（最终提交结果以该字段为准）。
- `Question`（可选）：题目文本，便于评审快速核对题意与答案对应关系。
- `reasoning_trace`（可选）：Agent 推理轨迹/解题步骤，建议按步骤记录关键检索、判断和计算过程。

关于可选字段补充说明：
- `Question` 和 `reasoning_trace` 均为可选，不影响基础提交有效性。
- 建议尽量提供 `reasoning_trace`，便于定位方案优劣与改进方向。在同等正确率前提下，`reasoning_trace` 内容完整、步骤清晰、可复现性强的提交可作为评审加分项。

其他建议：
- 源码压缩包中建议保留解题过程日志（如工具调用记录、运行日志、关键中间结果）作为评审参考。
